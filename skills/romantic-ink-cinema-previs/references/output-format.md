# Output Format

Use this format when the user asks for a structured previs response.

## Beat Breakdown Format

```text
0:00–0:02

Lens: [lens choice]
Camera: [camera motion]
Character Motion: [body choreography]
Energy Interaction: [how light/ink behaves]
Spatial Emotion: [distance / pressure / release]
Visual Density: [what fills the frame and how]
SFX: [sound texture]

0:02–0:04

Lens: ...
Camera: ...
Character Motion: ...
Energy Interaction: ...
Spatial Emotion: ...
Visual Density: ...
SFX: ...
```

## Short Prose Variant

Use this when the user wants something more elegant than a field list:

```text
0:00–0:02

[2–4 sentences combining lens, camera, motion, and energy into one cinematic paragraph]

SFX: [one concise sound line]
```

Rules:

- Keep one block per beat.
- Keep lens explicit whenever shot language matters.
- Always include SFX when timing is present.
- If the sequence is under 8 seconds, compress to 2-second or 1.5-second beats.

## System Summary Format

Use this when the user wants a reusable framework instead of a scene:

```text
Core Principle
[short statement]

Narrative Motion Formula
1. [stage]
2. [stage]
3. [stage]

Visual Language Rules
1. [rule]
2. [rule]
3. [rule]

Camera Choreography Pattern
[grouped lens/camera language]

Energy System
[variant + behavior]

SFX Structure
[audio logic]

Reusable Prompt Skeleton
[template]
```
